Why This Organization Works:

Buttons (Polaris Pattern)

Following Polaris button hierarchy: Primary for main actions, Secondary for alternative paths, Tertiary for low-priority actions, and Plain for subtle interactions. Buttons should be clear and predictable.

Polaris Best Practice: Use only ONE primary button per section to avoid overwhelming users. Group related actions with button groups.

Button Variants

Primary
Use to highlight the most important action. Don't use more than one per section to avoid overwhelming users. Example: "Save", "Submit", "Create Product"
Secondary
Use for important but not primary actions. Can be paired with primary for alternative paths. Example: "Cancel", "Discard", "Go Back"
Tertiary
Use for tertiary actions. Can be paired with any other button style. Usually appears as an underlined link. Example: "Learn more", "View details"
Plain
Use for less important or less commonly used actions. Used as actions in cards or for inline actions. Example: "Edit", "Remove", "Duplicate"

Button Sizes (Polaris)

Size Guidelines:
• Micro: Compact spaces, inline actions
• Slim: Tables, data grids, toolbars
• Medium: Default for most use cases
• Large: Main CTA in empty states or with large illustrations

Critical Actions (Destructive)

Critical Tone: Use when the action will delete data or be otherwise difficult to recover from. Always confirm before executing critical actions.

Design Principles

🌬️ Light & Airy
Generous white space, soft colors, and gentle shadows create breathing room. Nothing feels cramped or overwhelming.
🤝 Accessible First
Large text (17px base), high contrast (WCAG AAA), generous touch targets (56px min), clear visual hierarchy.
😌 Calming Yet Action-Oriented
Soft teal creates calm while warm coral accents drive action. Balance between soothing and motivating.
💚 Human & Friendly
Rounded corners, friendly language, approachable illustrations. Technology that feels warm and inviting.
📱 Mobile-First
Designed for thumbs and fingers, not mice. Large targets, simple navigation, optimized for one-handed use.
📊 Clarity Over Complexity
Show what matters, hide what doesn't. Progressive disclosure. Simple, scannable layouts with clear hierarchy.

External Mobile Design (65+ Audience)

Age-Friendly Design: These guidelines ensure the mobile experience works beautifully for users 65+
Typography
• Minimum 17px body text, 18px for important content
• Line height 1.6 minimum for easy reading
• High contrast text (4.5:1 minimum, AAA preferred)
• Avoid all-caps for long text (harder to read)
Touch Targets
• Minimum 44x44px, recommended 56x56px
• 8px minimum spacing between adjacent interactive elements
• Large, obvious tap areas for primary actions
• Avoid gesture-only interactions (provide buttons too)
Navigation
• Simple, clear navigation with 3-5 main sections
• Bottom navigation bar for one-handed use
• Large, labeled icons (not icon-only)
• Clear "back" affordances on every screen
Content
• One primary action per screen when possible
• Short paragraphs (2-3 sentences max)
• Bullet points over long text blocks
• Clear headings and scannable layouts
Feedback & States
• Immediate visual feedback on all interactions
• Clear loading states (avoid long waits)
• Success confirmations for important actions
• Forgiving UI (easy undo, confirmation dialogs)

Component Usage Guidelines

Button Hierarchy (Inspired by Function & Hinge Health)
Primary (Filled Blue): Main action, forward progress, completion, submission. The most important action on the page.
Secondary (Outlined Blue): Alternative actions, less critical paths, secondary workflows. Uses same brand color but with less visual weight.
Tertiary (Text Link Blue): Low-priority actions, back navigation, inline links, dismissals. Text-only, no background or border.
Destructive (Red): Only for actual destructive actions like delete, remove, cancel subscription. Not for general secondary actions.
Card elevation hierarchy
Standard: Default for most content grouping
Elevated: Interactive cards, clickable content, important callouts
Bordered: Dense layouts, nested cards, subtle grouping
Color Application (Minimal Approach)
Kivo Blue (#2B79B9): Your primary brand color. Use for all primary actions, links, focus states, interactive elements, and brand moments. This creates consistency and trust.

Neutrals (Grays): Use for text hierarchy, backgrounds, borders, and non-interactive UI. Most of your interface should be neutral.

Status Colors: Green (success), Orange (warning), Red (error/destructive), Blue (info). Only use for actual status communication.

Accent Colors (Use Sparingly):
• Gold (#F4B942): Achievements, milestones, celebrations only
• Peach (#FFB088): Subtle highlights in data visualizations

Following Function Health and Hinge Health's minimal approach: one strong brand color + neutrals + functional status colors.
