Actions

Components for user actions and interactions

Buttons
Primary way for users to take action. Use Primary for main action, Secondary for alternatives, Tertiary for low-priority, Plain for subtle actions.
Polaris Pattern: Use only ONE primary button per section. Group related actions with button groups.
<button class="btn btn-primary">Save Changes</button> <button class="btn btn-secondary">Cancel</button>
Button Sizes
Four sizes for different contexts: Micro (inline), Slim (tables), Medium (default), Large (CTAs)
When to use: Micro for inline actions, Slim for data tables/toolbars, Medium (default) for most cases, Large for empty states or main CTAs
Button Group
Group related actions together for clear visual hierarchy
Best Practice: Put the most important action first (left-aligned). Use consistent spacing (8px) between buttons.
Page Actions
Primary and secondary actions for a page, typically in the page header
Patient Details
Pattern: Page title on left, primary action on right. Secondary/destructive actions to the left of primary.

Animation Principles

‚ö°
Fast

Reduced Motion (prefers-reduced-motion)
Accessibility for users sensitive to motion
Why It Matters
Some users experience vestibular disorders, motion sickness, or ADHD that make animations uncomfortable or distracting. Respecting prefers-reduced-motion is a WCAG 2.1 Level AAA requirement.
Implementation
/* Default: Animations enabled */
.modal {
  animation: fadeIn 200ms ease-out;
}

/* Reduced motion: Disable or simplify */
@media (prefers-reduced-motion: reduce) {
  .modal {
    animation: none;
    /* Or use instant transition */
    transition: opacity 0ms;
  }

  /* Disable all transitions globally */
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
Best Practice: When reduced motion is enabled:
  • Disable decorative animations entirely (spinners, shimmers)
  • Keep essential feedback (success checkmark can still appear instantly)
  • Use instant transitions instead of removing them completely
  • Position changes are OK, just skip the smooth transition
Testing: Enable "Reduce Motion" in macOS Accessibility settings or Chrome DevTools > Rendering > Emulate CSS media prefers-reduced-motion.
