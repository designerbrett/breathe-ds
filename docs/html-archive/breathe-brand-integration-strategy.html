Breathe Brand Integration Strategy

Bridging Kivo Health's existing brand with the new Breathe design system for mobile and internal applications

Executive Summary

The Opportunity

Your existing Kivo Health brand already uses teal/turquoise as a primary brand color, creating a natural bridge to the Breathe design system. By maintaining brand continuity while optimizing for different audiences (65+ mobile users vs. internal teams), we can create cohesive yet purpose-built experiences.

Recommendation: Dual-Mode Design System

Implement a single design system with two application modes—External (Patient-Facing Mobile) and Internal (Dashboard/Tools)—that share core brand elements but optimize typography, density, and interaction patterns for their respective audiences.

Minimal Color Philosophy (Function Health & Hinge Health Approach)

One Strong Brand Color: Kivo Blue is used for all interactive elements, creating consistency and trust. Outlined buttons for secondary actions use the same blue, just with less visual weight.

No Secondary Button Colors: Unlike the initial proposal, we're NOT using coral/red for secondary buttons. This was confusing (looked like warnings). Instead, secondary buttons are outlined blue.

Accent Colors Used Sparingly: Gold for achievements/celebrations. Peach for subtle data viz highlights. Status colors (green/orange/red) only for actual status communication.

Updated Recommendation

Stick with Lexend + Inter for consistency across all Kivo Health properties. Lexend is designed for readability and accessibility (originally created for dyslexic readers), making it perfect for your 65+ audience. This maintains brand continuity between kivohealth.com, Breathe mobile apps, and internal tools.

Why Lexend Works Better Than Crimson Pro

Brand Continuity: Users who visit kivohealth.com and then download your mobile app will see consistent typography, reinforcing trust and professionalism.

Accessibility First: Lexend was specifically designed to improve reading speed and comprehension. Its letter forms reduce visual confusion—critical for older adults.

Modern Warmth: Lexend is softer than traditional sans-serifs but more modern than serifs, hitting the sweet spot of approachable yet professional.

Detailed Application Guidelines

External: Patient-Facing Mobile Apps (65+ Audience)

Primary Goals

Accessibility, clarity, encouragement, and ease of use. Every interaction should feel supportive and unintimidating.

  • Typography: 17-18px body text minimum. Headings at 24px+. Avoid all-caps for long text. Line height 1.6+.
  • Touch Targets: All buttons/interactive elements 56px minimum height. 8px spacing between adjacent tap targets.
  • Layout: Single-column layouts. One primary action per screen. Minimal scrolling required.
  • Navigation: Bottom tab bar (3-5 items max) with icons + labels. Clear back button on every sub-screen.
  • Color Usage: Generous use of white space. Teal for primary actions. Warm coral for encouragement. Gold for achievements.
  • Forms: One question at a time when possible. Large input fields (56px height). Clear helper text below each field.
  • Feedback: Immediate, obvious feedback on all interactions. Success animations. Clear error messages in plain language.
  • Content: Short paragraphs (2-3 sentences). Bullet points over prose. Clear headings. Minimal jargon.

Internal: Dashboards, Tables, Admin Tools (25-45 Tech-Savvy Staff)

Primary Goals

Efficiency, data density, productivity, and professional aesthetic. Enable staff to work quickly and make informed decisions.

  • Typography: 14-15px body text. Tighter line-height (1.5). Can use smaller labels (12-13px) for data tables.
  • Click Targets: 36px minimum for buttons. Can use smaller interactive elements (checkboxes, dropdowns) with precision mouse control.
  • Layout: Multi-column layouts. Sidebar navigation. Dense data tables with sticky headers. Efficient use of vertical space.
  • Navigation: Left sidebar with collapsible sections. Breadcrumbs for deep navigation. Quick actions in header.
  • Color Usage: Professional restraint. Teal for primary actions. Use status colors functionally (success, warning, error).
  • Forms: Multi-column forms. Inline validation. Field groups. Progressive disclosure for complex workflows.
  • Data Visualization: Complex charts with tooltips, legends, filters. Multiple data series. Hover interactions.
  • Tables: Sortable columns. Inline editing. Bulk actions. Pagination. Row selection. Filtering.
  • Efficiency Features: Keyboard shortcuts. Batch operations. Quick filters. Search with autocomplete. Saved views.

Implementation Roadmap

Phase 1: Foundation (Weeks 1-2)

Establish Core System

Typography: Switch from Crimson Pro to Lexend for brand consistency. Define type scales for both external and internal modes.

Color Tokens: Create design tokens for all colors. Maintain teal as primary across both modes. Define when to use coral, gold, and status colors.

Spacing System: Establish 8px base grid with larger spacing multipliers for external vs. internal.

Phase 2: Component Library (Weeks 3-5)

Build Dual-Mode Components

External Variants: Large buttons, spacious cards, simplified forms, bottom navigation, large type.

Internal Variants: Compact buttons, dense tables, multi-column forms, sidebar navigation, data visualization components.

Shared Components: Badges, alerts, modals, tooltips that work across both modes with size/spacing adjustments.

Phase 3: Documentation & Handoff (Week 6)

Enable Teams

Design Documentation: Clear guidelines on when to use external vs. internal specs. Component usage examples. Accessibility checklist.

Development Handoff: Design tokens (JSON/CSS variables). Component library in React (for Mantine integration). Storybook or similar for component exploration.

Testing: User testing with 65+ participants for mobile apps. Internal staff feedback on dashboard/tools.

Mantine Integration Strategy

Your internal systems use Mantine. Here's how to integrate Breathe with Mantine while keeping mobile apps independent.

Recommendation: Theme Mantine, Build Mobile Custom

Internal Tools: Keep using Mantine but override the theme with Breathe design tokens (colors, typography, spacing, border radius). This gives you Mantine's robust component library with Breathe's aesthetic.

Mobile Apps: Build custom components optimized for 65+ users. Mantine's components are desktop-focused and won't give you the large touch targets, simplified interactions, and generous spacing needed for seniors.

Mantine Theme Override Example

Override Mantine's default theme with Breathe tokens:

const breatheTheme = {
  colorScheme: 'light',
  colors: {
    brand: ['#E8F8F7', '#C4F1ED', '#9EEAE3', ..., '#4ECDC4', '#3DB8AF'],
    // Teal scale for primary color
  },
  primaryColor: 'brand',
  fontFamily: 'Inter, sans-serif',
  headings: { fontFamily: 'Lexend, sans-serif' },
  spacing: { xs: 8, sm: 12, md: 16, lg: 24, xl: 32 },
  radius: { sm: 8, md: 12, lg: 16 },
  defaultRadius: 'md',
};

Final Recommendations Summary

The Bridge Strategy

Create one Breathe design system with two application modes that share DNA (colors, typography, principles) but optimize for different contexts and users.

  • Colors: Keep Breathe Teal (#4ECDC4) as primary to align with existing Kivo Health brand. Extend with warm accent colors for mobile emotion and internal functionality.
  • Typography: Switch to Lexend + Inter (from Crimson Pro + Inter) to maintain consistency with kivohealth.com and leverage Lexend's accessibility benefits.
  • External Mode: Optimize for 65+ mobile users with large text (17px+), generous touch targets (56px), low density, and encouraging tone.
  • Internal Mode: Optimize for tech-savvy staff with efficient layouts, data density, professional tone, and productivity features.
  • Mantine: Theme Mantine with Breathe tokens for internal tools. Build custom components for mobile apps where Mantine doesn't fit.
  • Documentation: Create clear guidelines showing when to use external vs. internal specs, with component examples for each mode.
